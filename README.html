Notes for those working on the repo

Using pre-commit

Run pre-commit install to install the hooks. You now wont be able to commit until you pass the hooks. These (among other things) automatically format files and prevent us from committing ugly code. For more details, see the main docs and the R docs.

Using renv

renv maintains consistency between users R environments. Run renv::restore() and the environment will be downloaded into the repository based on the renv.lock file. If you want to add a packages to the lockfile, install the package and then run renv::snapshot(). For more details, see the docs.

Using Quarto for presentations

Quarto is pretty cool. I wont bore you, but have a look at the docs. Here, were using it for presentations. Its designed by the folks at RStudio, so you R folk will be happy. Make a .qmd file and run quarto render *.qmd to generate the html, which you can open in browser. We can get fancy and import our own css to have a consistent theme for out presentations.
